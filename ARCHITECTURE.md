## 数据流

### 1. 消息发送流程

```
用户输入 → UI 模块 → 消息加密 (加密模块) → 签名 (身份模块)
    → 信令封装 (信令模块) → 传输 (传输模块) → 对等节点
```

### 2. 文件传输流程

```
选择文件 → 文件分块 (文件传输模块) → Merkle 树构建
    → 多源调度 → 块传输 (传输模块) → 完整性验证 → 合并文件
```

### 3. 音视频通话流程

```
设备初始化 (媒体模块) → 编解码 → RTP 封装
    → 信令交换 (信令模块) → 传输建立 (传输模块)
    → 流媒体传输 → 质量监控 → 通话管理
```

### 4. 身份验证流程

```
创建身份 (身份模块) → 密钥生成 → Peer ID 生成
    → 加密存储 (存储模块) → 注册 (信令模块) → 在线
```

## 通信协议

### 信令协议

```
┌────────────────────────────────────────────┐
│          SignalingMessage                  │
├────────────────────────────────────────────┤
│ Type: offer/answer/candidate/invite/bye   │
│ From: Peer ID (发送方)                     │
│ To:   Peer ID (接收方)                     │
│ Data: SDP/ICE 候选/其他数据                │
│ Timestamp: 时间戳                          │
│ Signature: 数字签名                        │
│ Nonce: 防重放随机数                        │
└────────────────────────────────────────────┘
```

### 传输协议

```
┌────────────────────────────────────────────┐
│              QUIC/WebRTC                   │
├────────────────────────────────────────────┤
│ 多路复用流                                  │
│ 可靠/不可靠传输                            │
│ 拥塞控制                                   │
│ 连接迁移                                   │
└────────────────────────────────────────────┘
```

### 文件格式

```
┌────────────────────────────────────────────┐
│            FileMetadata                    │
├────────────────────────────────────────────┤
│ FileID:    Merkle 根哈希                   │
│ FileName:  文件名                           │
│ FileSize:  文件大小                        │

│ MerkleRoot: Merkle 树根                    │
└────────────────────────────────────────────┘
```

## 安全架构

### 加密层次

```
┌─────────────────────────────────────────────┐
│           应用层：消息加密                    │
│           AES-GCM-256                       │
├─────────────────────────────────────────────┤
│           传输层：TLS/DTLS                   │
│           QUIC/WebRTC 内置加密               │
├─────────────────────────────────────────────┤
│           身份层：数字签名                    │
│           Ed25519                           │
├─────────────────────────────────────────────┤
│           密钥层：密钥交换                    │
│           X25519 ECDH                       │
└─────────────────────────────────────────────┘
```

### 密钥管理

```
主密钥 (用户密码派生)
    │
    ├─→ 身份密钥 (Ed25519)
    │      └─→ 用于消息签名
    │
    ├─→ 加密密钥 (AES-256)
    │      └─→ 用于本地数据加密
    │
    └─→ 会话密钥 (X25519)
           └─→ 用于 P2P 通信加密
```

## 性能优化

### 缓存策略

```
┌─────────────────────────────────────────────┐
│              L1: 内存缓存                    │
│           LRU 策略，快速访问                  │
├─────────────────────────────────────────────┤
│              L2: SQLite 缓存                 │
│           持久化，中等速度                    │
├─────────────────────────────────────────────┤
│              L3: 文件系统                    │
│           大容量，慢速访问                    │
└─────────────────────────────────────────────┘
```

### 并发模型

```
主协程 (UI/CLI)
    │
    ├─→ 信令协程 (WebSocket)
    │
    ├─→ 传输协程 (QUIC/WebRTC)
    │
    ├─→ 文件传输协程 (多源下载)
    │
    └─→ 媒体协程 (音视频处理)
```

## 扩展性设计

### 插件架构

```
核心系统
    │
    ├─→ 传输插件 (QUIC/WebRTC/其他)
    │
    ├─→ 编解码插件 (Opus/VP8/H.264/其他)

    └─→ 存储插件 (SQLite/其他数据库)
```
